# Unit 3: Loops - Sensory Symphony

## Workshop Overview

**Team size:** Four students  
**Slide source:** Use the named slides/steps from the official LEGO facilitation-note print/PDF pages embedded in the agendas below.

The core combines B111-B114. Hour 1 compares forever and counted repeat loops through dragon scenarios. Hour 2 debugs a waterslide and coordinates scripts with messages. The optional extension adapts both parts of B115 Sensory Symphony into an accessible percussion-performance challenge.

## Facilitation Notes Reference

| Lesson | Context | Learning Outcome Or Skill | Student Prompt |
| --- | --- | --- | --- |
| [B111 Gold Patrol](https://teach.legoeducation.com/en-gb/computer-science/lesson/gold-patrol/facilitation-notes) | A dragon patrols treasure continuously. | Create a `forever` loop and explain simplification. | What stops a forever loop? |
| [B112 Dragon Defence](https://teach.legoeducation.com/en-gb/computer-science/lesson/dragon-defense/facilitation-notes) | A dragon repeats a behavior while protecting gold. | Use and tune a counted `repeat` loop. | What if the repeat count is too high or low? |
| [B113 Winning Waterslide](https://teach.legoeducation.com/en-gb/computer-science/lesson/winning-waterslide/facilitation-notes) | A slide releases riders and resets. | Debug and develop a loop-driven program. | What should happen continuously? |
| [B114 Slide Signal](https://teach.legoeducation.com/en-gb/computer-science/lesson/slide-signal/facilitation-notes) | A signal tells the next rider when it is safe to go. | Send and receive messages across scripts. | When is a message sent? |
| [B115 Sensory Symphony](https://teach.legoeducation.com/en-gb/computer-science/lesson/sensory-symphony/facilitation-notes) | Design percussion that can be seen, heard and felt. | Use loops and design for accessibility. | How can more people experience the performance? |

## Team Roles

Rotate designer, builder, programmer and rehearsal tester at `1:00`, after lunch and at extension `1:00`.

## Adapted B115 Rubric

| Criterion | Workshop Target | Stretch |
| --- | --- | --- |
| Loops | Modify a loop to change a repeating beat. | Create a new repeating beat. |
| Accessibility | Describe one accessibility improvement. | Explain multiple improvements. |
| Hardware and software | Use software to control hardware. | Create and modify the control program. |
| Prototype | Create a percussion performance. | Iterate to make it more accessible. |
